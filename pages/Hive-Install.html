系统环境

CentOS 7
Hadoop 2.2.0
JDK 1.8

环境变量

需要设置HIVE_HOME，HADOOP_HOME，编辑/etc/profile文件

  1. # java和hadoop的环境变量在安装相应软件时已配置
  2. # java config
  3. export JAVA_HOME=/usr/java/jdk1.8.0_65
  4. export JRE_HOME=${JAVA_HOME}/jre
  5. export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
  6. export PATH=${JAVA_HOME}/bin:$PATH
  8. # hadoop config
  9. export HADOOP_HOME=/home/hadoop/hadoop-2.2.0
  10. export PATH=$HADOOP_HOME/bin:$PATH
  12. # hive config
  13. export HIVE_HOME=/home/hadoop/hive-0.12.0
  14. export PATH=${HIVE_HOME}/bin:${HIVE_HOME}/conf:$PATH

配置文件

在hive安装目录中/home/hadoop/hive-0.12.0/conf，创建hive-site.xml和hive-env.sh

  1. mv hive-site.xml.template hive-site.xml
  2. mv hive-env.sh.template hive-env.sh

修改配置文件-1

  1. # 针对hive-env.sh文件，需要修改java_home
  2. HADOOP_HOME=/home/hadoop/hadoop-2.2.0

修改配置文件-2

  1. # 修改hive-site.xml文件
  2. # hive需要将元数据存储在关系型数据库中，默认使用Derby
  3. # 一般而言而言会安装mysql进行替换，以下配置以mysql为例
  1. # 针对mysql连接的配置
  2. <property>
  3.   <name>javax.jdo.option.ConnectionURL</name>
  4.   <value>jdbc:mysql://hadoop-master:3306/hivedb?createDatabaseIfNotExist=true</value>
  5.   <description>JDBC connect string for a JDBC metastore</description>
  6. </property>
  8. <property>
  9.   <name>javax.jdo.option.ConnectionDriverName</name>
  10.   <value>com.mysql.jdbc.Driver</value>
  11.   <description>Driver class name for a JDBC metastore</description>
  12. </property>
  14. <property>
  15.   <name>javax.jdo.option.ConnectionUserName</name>
  16.   <value>hive</value>
  17.   <description>username to use against metastore database</description>
  18. </property>
  20. <property>
  21.   <name>javax.jdo.option.ConnectionPassword</name>
  22.   <value>hive</value>
  23.   <description>password to use against metastore database</description>
  24. </property>
  1. # 针对hive status的配置
  2. <property>
  3.   <name>hive.stats.dbclass</name>
  4.   <value>jdbc:mysql</value>
  5.   <description>The default database that stores temporary hive statistics.</description>
  6. </property>
  8. <property>
  9.   <name>hive.stats.jdbcdriver</name>
  10.   <value>com.mysql.jdbc.Driver</value>
  11.   <description>The JDBC driver for the database that stores temporary hive statistics.</description>
  12. </property>
  14. <property>
  15.   <name>hive.stats.dbconnectionstring</name>
  16.   <value>jdbc:mysql://hadoop-master:3306/hive_status?createDatabaseIfNotExist=true</value>
  17.   <description>The default connection string for the database that stores temporary hive statistics.</description>
  18. </property>
  1. # 针对metastore的配置，避免出现不能初始化metastore的问题
  2. <property>
  3.   <name>hive.metastore.authorization.storage.checks</name>
  4.   <value>false</value>
  5.   <description>Should the metastore do authorization checks against the underlying storage
  6.   for operations like drop-partition (disallow the drop-partition if the user in
  7.   question doesn't have permissions to delete the corresponding directory
  8.   on the storage).</description>
  9. </property>

mysql和hadoop配置

mysql配置

由于hive要在mysql中建立表，需要将结果写入hadoop中，因此需要对应的配置

  1. --针对mysql的配置
  2. --主要是创建专用用户，赋予权限
  3. --为hive用户赋予所有权限
  4. grant all privileges on *.* to 'hive'@'hadoop-master' with grant option;
  5. --强制写入
  6. flush privileges;
  7. --查看用户权限结果
  8. SELECT DISTINCT CONCAT('User: ''',user,'''@''',host,''';') AS query FROM mysql.user;
  10. --结果如下
  11. +-------------------------------+
  12. | query                         |
  13. +-------------------------------+
  14. | User: 'hive'@'%';             |
  15. | User: 'root'@'127.0.0.1';     |
  16. | User: 'root'@'::1';           |
  17. | User: ''@'hadoop-master';     |
  18. | User: 'hive'@'hadoop-master'; |
  19. | User: 'root'@'hadoop-master'; |
  20. | User: ''@'localhost';         |
  21. | User: 'hive'@'localhost';     |
  22. | User: 'root'@'localhost';     |
  23. +-------------------------------+

除了mysql用户创建之外，还需要将mysql-connector-java-5.1.38-bin.jar拷贝到/home/hadoop/hive-0.12.0/lib中。

hadoop配置

  1. # hadoop的配置主要是创建目录，用户hive存储数据和临时文件
  2. # 创建目录的位置为hive-site.xml中默认的位置，不需要再修改此文件
  3. $HADOOP_HOME/bin/hadoop fs -mkdir       /tmp
  4. $HADOOP_HOME/bin/hadoop fs -mkdir       /user/hive/warehouse
  5. $HADOOP_HOME/bin/hadoop fs -chmod g+w   /tmp
  6. $HADOOP_HOME/bin/hadoop fs -chmod g+w   /user/hive/warehouse

hive服务

首先需要启动metastore服务

hive --service metastore # 启动元数据存储服务

服务正常启动后，mysql中会新创建几张表。
进入到hive目录bin目录中直接输入hive启动服务，运行一下命令，结果正常，则说明hive安装成功。

  1. show databases;
  2. OK
  3. default
  4. Time taken: 2.544 seconds, Fetched: 1 row(s)
